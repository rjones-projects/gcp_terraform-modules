# Workload Identity for GitHub Actions

This module configures a Google Cloud Workload Identity Pool and Provider for
GitHub Actions and grants a service account the `roles/iam.workloadIdentityUser`
role using Vodafone-specific settings.

It is designed to be driven by `workload_identity.spec` from `terraform.tfvars.json`
generated by `yaml_to_tfvars.py` and `project.yaml`.

## Features

- Creates one Workload Identity Pool and Provider for GitHub Actions
- Uses Vodafone GitHub OIDC issuer and attribute mapping
- Grants `roles/iam.workloadIdentityUser` on a target service account
- Integrates with `project.yaml` via a `workload_identity` module block

## Usage with `project.yaml`

```yaml
iam_service_account:
  source: "../modules/iam_service_account"
  depends_on: []
  spec:
    - name: "github-actions"
      iam_project_roles:
        "PROJECT_ID":
          - "roles/iam.serviceAccountTokenCreator"

workload_identity:
  source: "../modules/workload_identity"
  depends_on:
    - iam_service_account
  spec:
    - service_account_email: "projects/PROJECT_ID/serviceAccounts/github-actions@PROJECT_ID.iam.gserviceaccount.com"
      github_org: "VFGROUP-NSE-DNOSS"
      github_repo: "DNE-DE-NGDI-SOURCECODE"
      pool_id: "github2-pool2"
      provider_id: "github2-provider2"
```

`yaml_to_tfvars.py` passes `workload_identity = var.workload_identity` in `deploy/main.tf`.
This module reads `var.workload_identity.spec[0]` in `locals.tf`.

## Minimal Terraform usage

```hcl
module "iam_service_account" {
  source     = "../modules/iam_service_account"
  project_id = var.project_id

  iam_service_account = {
    spec = [
      {
        name = "github-actions"
        iam_project_roles = {
          "${var.project_id}" = [
            "roles/iam.serviceAccountTokenCreator"
          ]
        }
      }
    ]
  }
}

module "workload_identity" {
  source     = "../modules/workload_identity"
  project_id = var.project_id
  region     = var.region

  workload_identity = {
    spec = [
      {
        service_account_email = module.iam_service_account.ids["github-actions"]
        github_org            = "VFGROUP-NSE-DNOSS"
        github_repo           = "DNE-DE-NGDI-SOURCECODE"
        pool_id               = "github2-pool2"
        provider_id           = "github2-provider2"
      }
    ]
  }
}
```

## Input

- `project_id` (`string`, required)
- `region` (`string`, optional compatibility input)
- `workload_identity` (`any`, expected to contain `spec` list)

Each `workload_identity.spec` item supports:

- `service_account_email` (string, required)
- `github_org` (string, default `"VFGROUP-NSE-DNOSS"`)
- `github_repo` (string, default `"DNE-DE-NGDI-SOURCECODE"`)
- `pool_id` (string, default `"github2-pool2"`)
- `provider_id` (string, default `"github2-provider2"`)
- `display_name` (string, optional)
- `description` (string, optional)

## Outputs

- `workload_identity_pool_id`
- `workload_identity_provider_id`
