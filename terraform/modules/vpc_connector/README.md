# Google Cloud VPC Access Connector

This module creates one or more **Serverless VPC Access Connectors** using
`vpc_connector.spec` from `terraform.tfvars.json` generated by `yaml_to_tfvars.py`.

It is designed to work together with the `network` module in this repository:
you define VPC networks and subnets with `network`, then point connectors at
those subnets.

## Usage with `project.yaml`

```yaml
project_id: "fluent-drummer-468119-c3"
region: "europe-central2"

terraform:
  required_version: ">= 1.14.0, < 2.0.0"
  providers:
    google: "~> 7.17.0"

finops_labels:
  source: "../modules/finops_labels"
  depends_on: []
  spec: []

network:
  source: "../modules/network"
  depends_on:
    - finops_labels
  spec:
    - name: "ngdi-main"
      finops_resource_type: "networking"
      labels:
        vf_ngdi_environment: "alpha"
        vf_ngdi_shared: "true"
        vf_ngdi_goal: "networking"
        vf_ngdi_location: "europe-central2"
      subnets:
        - name: "ingress"
          region: "europe-central2"
          ip_cidr_range: "10.8.0.0/24"

vpc_connector:
  source: "../modules/vpc_connector"
  depends_on:
    - network
  spec:
    - name: "ngdi-ingress-connector"
      region: "europe-central2"
      # IMPORTANT: this must match the subnet 'name' suffix created by the network module.
      # The actual subnetwork name will be "${common_resource_id}-subnet-${name}".
      subnet_name: "ingress"
      subnet_project_id: "fluent-drummer-468119-c3"
      min_instances: 2
      max_instances: 10
      machine_type: "e2-micro"
```

`yaml_to_tfvars.py` passes `vpc_connector = var.vpc_connector` in `deploy/main.tf`.
This module reads `var.vpc_connector.spec` in `locals.tf`.

## Minimal Terraform usage

```hcl
module "network" {
  source     = "../modules/network"
  project_id = var.project_id
  region     = var.region

  network = {
    spec = [
      {
        name                 = "ngdi-main"
        finops_resource_type = "networking"
      }
    ]
  }
}

module "vpc_connector" {
  source     = "../modules/vpc_connector"
  project_id = var.project_id
  region     = var.region

  vpc_connector = {
    spec = [
      {
        name             = "ngdi-ingress-connector"
        region           = var.region
        # Must match one of the subnet 'name' values used in the network module spec.
        subnet_name      = "ingress"
        subnet_project_id = var.project_id
      }
    ]
  }
}
```

## Inputs

Top-level input:

- `project_id` (`string`, required)
- `region` (`string`, optional, used as default connector region)
- `vpc_connector` (`any`, expected shape below)

Expected `vpc_connector.spec` item attributes:

- `name` (string, required) — connector name.
- `region` (string, optional) — falls back to `var.region` when omitted.
- `subnet_name` (string, required) — **suffix** of the subnet name used in the `network` module spec.
- `subnet_project_id` (string, optional) — project hosting the subnet; defaults to `project_id`.
- `min_instances` (number, optional).
- `max_instances` (number, optional).
- `machine_type` (string, optional).

## Outputs

- `connectors`: map of created VPC Access Connector resources.
- `connector_ids`: map `name => resource id`.
